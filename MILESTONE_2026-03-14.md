# 里程碑作战表（2026-03-14）

## 目标（D1 评审日必须达标）
- 关键路径闭环：大厅 → 建房/入房 → 准备/开局 → 至少一轮出牌/过牌 → 结算 → 再来一局/离开
- CI 全绿：web-checks / web-e2e / web-perf
- 失败可恢复：登录失败/断网/房间不存在/满员均有可操作兜底

## 角色与分工（Owner）
- TL：里程碑裁决、范围控制、阻塞升级与评审材料
- FE-1：房间页架构收敛、Realtime 稳定性与性能抖动治理
- FE-2：鉴权/匿名登录统一、可测性（data-testid）与 UX 一致性
- BE-1：RPC 契约/错误码、RLS 最小权限、关键写入幂等与并发冲突策略
- BE-2：联调与数据修复脚本、状态机与字段契约对齐
- QA-1：Smoke/回归编排、Flaky 治理与门禁策略
- QA-2：用例落地与报告（trace/screenshot/复现步骤）
- DevOps：CI/CD、环境/Secrets、产物留存、性能基线趋势

## 今日开工清单（D0）
### FE-1（P0）
- 拆分房间页组件（优先：Overlay/Seat/Hand/Table），降低单文件复杂度
- Realtime 拉取节流：避免成员变更导致连续 fetch/渲染风暴
- 验收：本地 smoke 流程可重复 3 次无卡死

### FE-2（P0）
- 收敛鉴权逻辑：Home/Lobby/Room 统一 ensureAuthed 入口
- 补齐关键 data-testid：手牌容器、出牌/过牌按钮、结算弹层、房间覆盖层关键按钮
- 验收：E2E 选择器不依赖 class，主流程可稳定跑通

### BE-1（P0）
- 冻结关键 RPC 契约：create/join/leave/start/turn，补齐可机读错误码
- RLS 最小权限基线：默认拒绝；越权负测稳定失败
- 验收：重复请求不重复落库；并发冲突返回明确错误码

### BE-2（P0）
- 联调关键 RPC：字段/状态机对齐，补齐必要数据修复/巡检脚本
- 验收：测试环境出现坏数据可在 5 分钟内恢复到可测

### QA-1（P0）
- 定义 Smoke E2E 2–3 条（核心闭环最短路径）并设为门禁候选
- Flaky 规则：失败即归因（环境/数据/等待/并发）并指派修复 Owner
- 验收：CI 上失败可复现可定位（trace/screenshot）

### QA-2（P0）
- 落地 Smoke 用例：练习房开局、PVP 创建加入、房间异常覆盖层
- 验收：连续跑 3 次通过率 100%（或隔离并提交修复任务）

### DevOps（P0）
- CI 产物留存：Playwright report/trace 作为 artifact（失败时必有）
- 性能基线：保留 web-perf 结果并形成可比对输出（先趋势后门禁）
- 验收：任一失败可在 10 分钟内定位到具体步骤
