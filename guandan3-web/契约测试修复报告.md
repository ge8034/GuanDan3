## 问题分析

### 原始问题
1. **测试跳过**: 所有契约测试都被跳过 (`describe.skipIf(!testApiUrl)`)
2. **依赖外部服务**: 测试需要运行中的API服务器 (`TEST_API_URL` 环境变量)
3. **测试框架不匹配**: 使用了不适合Next.js API Routes的测试方法

### 根本原因
契约测试配置为需要真实运行的HTTP服务器，这在测试环境中不可行。测试使用了 `describe.skipIf(!testApiUrl)` 条件跳过所有测试用例。

## 修复方案

### 1. 测试架构重构
**变更**: 从真实HTTP请求改为直接调用Next.js API路由处理器

```typescript
// 修复前：需要运行中的服务器
const testApiUrl = process.env.TEST_API_URL;
describe.skipIf(!testApiUrl)('健康检查 API 契约测试', () => {
  // 使用 fetch 发送HTTP请求
  const response = await fetch(`${testApiUrl}/api/health`);
});

// 修复后：直接导入和调用API处理器
import { GET as healthGET } from '@/app/api/health/route';

async function mockApiRequest() {
  const response = await healthGET();
  return {
    status: response.status,
    body: await response.json(),
    timings: { duration: endTime - startTime }
  };
}
```

### 2. 测试辅助函数优化
创建了 `mockApiRequest()` 辅助函数：
- 直接调用Next.js API路由处理器
- 测量响应时间用于性能测试
- 返回标准化的响应对象
- 无需外部服务器依赖
