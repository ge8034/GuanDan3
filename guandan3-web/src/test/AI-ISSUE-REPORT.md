# AI问题报告 - 2026-04-01

## 重大发现：AI不出牌问题

### 问题描述
在练习模式中，AI玩家没有自动出牌，游戏无法进行。

### 根本原因
`useAIDecision` Hook没有被触发，导致AI决策系统从未执行。

### 测试证据

#### E2E测试结果
| 测试项目 | 结果 | 详情 |
|---------|------|------|
| 创建房间 | ✅ 成功 | 1次create_practice_room API调用 |
| 开始游戏 | ✅ 成功 | 1次start_game API调用，返回204 |
| AI手牌查询 | ❌ 无 | 0次get_ai_hand API调用 |
| AI出牌 | ❌ 无 | 0次submit_turn API调用 |
| AI控制台日志 | ❌ 无 | 0条AI相关日志 |

### 问题分析

#### AI决策逻辑测试
- **246个单元测试全部通过** ✅
- AI决策函数`decideMove`工作正常
- 牌型识别、压牌判断逻辑正确

#### Hook触发测试
- **32个锁机制测试超时** ❌
- `useAIDecision` Hook可能没有被正确触发
- 需要检查Hook的依赖条件

### 可能原因

1. **游戏状态同步问题**
   - `gameStatus`可能没有从'deal'变成'playing'
   - `currentSeat`可能没有正确设置
   - `members`可能没有正确更新

2. **Hook触发条件问题**
   - `useAIDecision`需要以下条件：
     - `gameStatus === 'playing'`
     - `isOwner === true`
     - `currentMember.member_type === 'ai'`
   - 任一条件不满足都不会触发AI决策

3. **Realtime订阅问题**
   - 游戏状态变化可能没有被客户端订阅
   - 导致状态更新没有触发Hook重新执行

3. ⏳ 检查`useAIDecision`的触发条件
