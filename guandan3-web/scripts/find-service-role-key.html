🔑 如何找到 Supabase Service Role Key

1打开 Supabase Dashboard
https://supabase.com/dashboard

登录你的 Supabase 账户

2选择项目

在项目列表中找到你的项目：

项目名称 guandan3
项目 ID rzzywltxlfgucngfiznx
👇 点击进入项目
3进入设置页面

在左侧导航栏找到并点击：

⚙️ Settings （齿轮图标）
↓
API
4找到 Project API keys 区域

向下滚动，找到 Project API keys 部分

↓
Supabase Dashboard - API Settings
Project API keys
anon public eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOi...
← 这个是 anon key（不是你需要的）
⬇️ 继续往下看，找到第二个密钥
service_role eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYm...[更长]...8fQ
← 这个才是你需要的！
5复制 service_role 密钥

点击 service_role 右侧的 Copy 按钮

service_role eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3MiOiJzdXBhYmFzZSIsInJlZiI6InJ6enl3bHR4bGZndWNuZ2Zpem54Iiwicm9sZSI6InNlcnZpY2Vfcm9sZSIsImlhdCI6MTc2OTA1MzU2OSwiZXhwIjoyMDg0NjI5NTY5fQ.xxx...
注意：service_role key 通常比 anon key 更长（300+ 字符）
⚠️ 重要提示
  • anon public = 你之前复制的，以 eyJ...role":"anon" 结尾
  • service_role = 你需要的，以 eyJ...role":"service_role" 结尾
  • service_role key 必须完全复制，不要截断
6粘贴给我

复制完成后，将完整的 service_role key 粘贴给我，格式如下：

SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...[完整密钥]
✅ 验证方法

如果复制的密钥中包含 "role":"service_role"，那就是正确的！
