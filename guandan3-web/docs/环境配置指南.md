## 概述

本文档详细说明了掼蛋游戏在不同环境下的配置方法，包括开发环境、生产环境的搭建，以及Supabase、数据库、缓存等服务的配置。

### 环境类型

- **开发环境**：本地开发，用于开发和测试
- **测试环境**：模拟生产环境，用于集成测试
- **生产环境**：正式上线环境，面向真实用户

### 技术要求

#### 开发环境

- **操作系统**：Windows 10+、macOS 10.15+、Linux
- **Node.js**：18.x 或更高版本
- **npm**：9.x 或更高版本
- **Git**：2.x 或更高版本
- **浏览器**：Chrome 90+、Firefox 88+、Safari 14+

#### 生产环境

- **服务器**：Linux (Ubuntu 20.04+)
- **Node.js**：18.x LTS
- **内存**：至少 2GB RAM
- **存储**：至少 20GB 可用空间
- **网络**：稳定的互联网连接

### 4. 配置Nginx反向代理

#### 安装Nginx

```bash
# 安装Nginx
sudo apt-get install -y nginx

# 启动Nginx
sudo systemctl start nginx
sudo systemctl enable nginx
```

#### 配置Nginx

```bash
# 创建站点配置
sudo nano /etc/nginx/sites-available/guandan3
```

添加以下配置：

```nginx
upstream guandan3_backend {
    server 127.0.0.1:3000;
    keepalive 64;
}

server {
    listen 80;
    listen [::]:80;
    server_name your-domain.com www.your-domain.com;

    # 重定向到HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name your-domain.com www.your-domain.com;

    # SSL证书配置
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;

    # 日志
    access_log /var/log/nginx/guandan3_access.log;
    error_log /var/log/nginx/guandan3_error.log;

    # 静态文件缓存
    location /_next/static {
        alias /var/www/guandan3/guandan3-web/.next/static;
        expires 365d;
        add_header Cache-Control "public, immutable";
    }

    location /static {
        alias /var/www/guandan3/guandan3-web/public/static;
        expires 365d;
        add_header Cache-Control "public, immutable";
    }

    # Next.js应用
    location / {
        proxy_pass http://guandan3_backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
        proxy_connect_timeout 75s;
    }
}
```

#### 启用配置

```bash
# 创建符号链接
sudo ln -s /etc/nginx/sites-available/guandan3 /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载Nginx
sudo systemctl reload nginx
```

## 部署流程

### 自动部署（CI/CD）

#### GitHub Actions配置

创建`.github/workflows/deploy.yml`：

```yaml
name: Deploy to Production

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
    - name: Checkout code
      uses: actions/checkout@v3
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
    
    - name: Install dependencies
      run: npm ci
    
    - name: Run tests
      run: npm test
    
    - name: Build application
      run: npm run build
      env:
        NEXT_PUBLIC_SUPABASE_URL: ${{ secrets.SUPABASE_URL }}
        NEXT_PUBLIC_SUPABASE_ANON_KEY: ${{ secrets.SUPABASE_ANON_KEY }}
    
    - name: Deploy to server
      uses: easingthemes/ssh-deploy@v2.1.5
      env:
        SSH_PRIVATE_KEY: ${{ secrets.SSH_PRIVATE_KEY }}
        REMOTE_HOST: ${{ secrets.REMOTE_HOST }}
        REMOTE_USER: ${{ secrets.REMOTE_USER }}
        TARGET: /var/www/guandan3/guandan3-web
      with:
        SOURCE: ".next/"
    
    - name: Restart application
      uses: appleboy/ssh-action@master
      with:
        host: ${{ secrets.REMOTE_HOST }}
        username: ${{ secrets.REMOTE_USER }}
        key: ${{ secrets.SSH_PRIVATE_KEY }}
        script: |
          cd /var/www/guandan3/guandan3-web
          pm2 restart guandan3
```

#### 配置GitHub Secrets

1. 进入GitHub仓库Settings
2. 点击Secrets → New repository secret
3. 添加以下secrets：
   - `SUPABASE_URL`
   - `SUPABASE_ANON_KEY`
   - `SSH_PRIVATE_KEY`
   - `REMOTE_HOST`
   - `REMOTE_USER`

### 4. 备份策略

#### 数据库备份

```bash
# 创建备份脚本
cat > /var/www/guandan3/backup-db.sh << 'EOF'
#!/bin/bash

BACKUP_DIR="/var/backups/guandan3"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="guandan3_backup_${DATE}.sql"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 使用pg_dump备份
pg_dump -h db.xxx.supabase.co -U postgres -d postgres > $BACKUP_DIR/$BACKUP_FILE

# 压缩备份文件
gzip $BACKUP_DIR/$BACKUP_FILE

# 删除7天前的备份
find $BACKUP_DIR -name "guandan3_backup_*.sql.gz" -mtime +7 -delete

echo "Backup completed: $BACKUP_FILE.gz"
EOF

# 设置执行权限
chmod +x /var/www/guandan3/backup-db.sh

# 添加到cron（每天凌晨3点备份）
crontab -e
# 添加：0 3 * * * /var/www/guandan3/backup-db.sh
```

## 故障排除

### 常见问题

#### 1. 应用无法启动

**问题**：PM2启动失败

**解决方案**：
```bash
# 查看详细错误
pm2 logs guandan3 --err

# 检查端口占用
sudo netstat -tulpn | grep :3000

# 检查环境变量
pm2 env guandan3

# 重启应用
pm2 restart guandan3
```

#### 2. 数据库连接失败

**问题**：无法连接到Supabase

**解决方案**：
```bash
# 检查网络连接
ping your-project.supabase.co

# 检查环境变量
echo $NEXT_PUBLIC_SUPABASE_URL
echo $NEXT_PUBLIC_SUPABASE_ANON_KEY

# 测试数据库连接
psql -h db.xxx.supabase.co -U postgres -d postgres
```

#### 3. 内存不足

**问题**：应用因内存不足崩溃

**解决方案**：
```bash
# 增加swap空间
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 永久启用swap
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# 优化PM2配置
pm2 reload guandan3 --max-memory-restart 1G
```

#### 4. SSL证书问题

**问题**：SSL证书过期或配置错误

**解决方案**：
```bash
# 检查证书状态
sudo certbot certificates

# 手动续期
sudo certbot renew

# 重新配置Nginx
sudo nginx -t
sudo systemctl reload nginx
```

#### 5. 性能问题

**问题**：应用响应缓慢

**解决方案**：

### 2. SSH安全

```bash

sudo nano /etc/ssh/sshd_config

# 添加：PasswordAuthentication no
```
