## 前置要求

### 服务器要求

- **操作系统**: Ubuntu 20.04+ 或 CentOS 8+
- **CPU**: 2核以上
- **内存**: 4GB以上
- **磁盘**: 50GB以上SSD
- **网络**: 公网IP，开放80、443端口

### 软件要求

- **Node.js**: 18.x 或更高版本
- **npm**: 9.x 或更高版本
- **PM2**: 最新版本
- **Nginx**: 1.18+ 或更高版本
- **Git**: 2.x 或更高版本

### 域名和SSL

- **域名**: 已注册的域名（如 guandan3.com）
- **SSL证书**: Let's Encrypt 免费证书

## 环境准备

### 1. 安装Node.js

```bash
# 使用NodeSource仓库安装Node.js 18.x
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证安装
node -v
npm -v
```

### 2. 安装PM2

```bash
# 全局安装PM2
sudo npm install -g pm2

# 验证安装
pm2 -v
```

### 3. 安装Nginx

```bash
# 安装Nginx
sudo apt-get update
sudo apt-get install -y nginx

# 验证安装
nginx -v
```

### 4. 安装Certbot（SSL证书）

```bash
# 安装Certbot
sudo apt-get install -y certbot python3-certbot-nginx

# 获取SSL证书
sudo certbot --nginx -d guandan3.com -d www.guandan3.com

# 设置自动续期
sudo certbot renew --dry-run
```

### 5. 克隆代码仓库

```bash
# 创建应用目录
sudo mkdir -p /var/www/guandan3-web
sudo chown -R $USER:$USER /var/www/guandan3-web

# 克隆代码
cd /var/www/guandan3-web
git clone https://github.com/your-username/guandan3-web.git .
```

## 应用部署

### 1. 配置环境变量

```bash
# 复制环境变量模板
cp .env.production.example .env.production

# 编辑环境变量
nano .env.production
```

**重要配置项**:

```bash
# Supabase配置
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here

# 应用配置
NEXT_PUBLIC_APP_URL=https://guandan3.com
NEXT_PUBLIC_APP_NAME=掼蛋3
NEXT_PUBLIC_APP_VERSION=1.0.0

# Sentry错误监控
NEXT_PUBLIC_SENTRY_DSN=https://your-sentry-dsn@sentry.io/project-id
SENTRY_ORG=your-sentry-org
SENTRY_PROJECT=guandan3
SENTRY_AUTH_TOKEN=your-sentry-auth-token
```

### 2. 安装依赖

```bash
# 安装生产依赖
npm ci --production=false
```

### 3. 构建应用

```bash
# 构建生产版本
npm run build
```

### 4. 配置PM2

```bash
# 创建日志目录
mkdir -p logs

# 启动应用
pm2 start ecosystem.config.js --env production

# 保存PM2配置
pm2 save

# 设置开机自启
pm2 startup | tail -n 1 | bash
```

### 5. 验证应用运行

```bash
# 查看应用状态
pm2 status

# 查看应用日志
pm2 logs guandan3-web

# 测试应用
curl http://localhost:3000/health
```

## Nginx配置

### 1. 复制Nginx配置

```bash
# 复制配置文件
sudo cp nginx.conf /etc/nginx/sites-available/guandan3

# 创建软链接
sudo ln -s /etc/nginx/sites-available/guandan3 /etc/nginx/sites-enabled/

# 删除默认配置
sudo rm /etc/nginx/sites-enabled/default
```

### 2. 测试Nginx配置

```bash
# 测试配置文件
sudo nginx -t
```

### 3. 重启Nginx

```bash
# 重启Nginx
sudo systemctl restart nginx

# 查看Nginx状态
sudo systemctl status nginx
```

### 4. 配置防火墙

```bash
# 允许HTTP和HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 启用防火墙
sudo ufw enable

# 查看防火墙状态
sudo ufw status
```


### 2. 配置PM2监控

```bash
# 安装PM2 Logrotate
pm2 install pm2-logrotate

# 配置日志轮转

# 配置监控阈值
pm2 set pm2-server-monit:cpu 80
pm2 set pm2-server-monit:memory 80
pm2 set pm2-server-monit:disk 80
```

## 故障排查

### 应用无法启动

**症状**: PM2显示应用状态为errored

**解决方案**:

```bash
# 查看错误日志
pm2 logs guandan3-web --err

# 检查端口占用
sudo netstat -tlnp | grep 3000

# 检查环境变量
cat .env.production

# 重新启动应用
pm2 restart guandan3-web
```

### Nginx 502错误

**症状**: 访问网站显示502 Bad Gateway

**解决方案**:

```bash
# 检查Nginx配置
sudo nginx -t

# 检查Nginx日志
sudo tail -f /var/log/nginx/error.log

# 检查应用是否运行
pm2 status

# 重启Nginx
sudo systemctl restart nginx
```

### SSL证书问题

**症状**: HTTPS无法访问或证书过期

**解决方案**:

```bash
# 手动续期证书
sudo certbot renew

# 强制续期
sudo certbot renew --force-renewal

# 重新加载Nginx
sudo systemctl reload nginx
```

### 数据库连接失败

**症状**: 应用无法连接到Supabase

**解决方案**:

```bash
# 检查环境变量
echo $NEXT_PUBLIC_SUPABASE_URL
echo $NEXT_PUBLIC_SUPABASE_ANON_KEY

# 测试数据库连接
./scripts/database-monitor.sh

# 检查Supabase状态
# 访问 https://supabase.com/dashboard
```

### 内存不足

**症状**: 应用频繁重启或响应缓慢

**解决方案**:

```bash
# 查看内存使用
free -h

# 查看PM2内存使用
pm2 monit

# 增加swap空间
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile

# 永久启用swap
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

## 安全建议

### 1. 定期更新系统

```bash
# 更新系统包
sudo apt-get update
sudo apt-get upgrade -y

# 更新Node.js
sudo npm install -g n
sudo n stable
```

### 2. 配置防火墙

```bash
# 只开放必要端口
sudo ufw allow 22/tcp  # SSH
sudo ufw allow 80/tcp  # HTTP
sudo ufw allow 443/tcp # HTTPS
sudo ufw enable
```

### 3. 配置SSH安全

```bash
# 禁用root登录
sudo sed -i 's/PermitRootLogin yes/PermitRootLogin no/' /etc/ssh/sshd_config

# 禁用密码登录（使用密钥）
sudo sed -i 's/PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config

# 重启SSH服务
sudo systemctl restart sshd
```

### 4. 定期备份

```bash
# 查看定时任务
crontab -l

# 手动执行备份
./scripts/backup-database.sh
./scripts/backup-application.sh
./scripts/backup-logs.sh
```
