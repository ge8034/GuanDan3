## 概述

本文档描述了掼蛋游戏的数据库设计，包括表结构、索引、安全策略和存储过程。数据库基于PostgreSQL，使用Supabase作为托管平台。

### 技术栈

- **数据库引擎**：PostgreSQL 15+
- **托管平台**：Supabase
- **安全机制**：Row Level Security (RLS)
- **实时功能**：Supabase Realtime
- **数据类型**：JSONB用于复杂数据存储

### 设计原则

1. **数据安全**：使用RLS确保用户只能访问自己的数据
2. **性能优化**：合理使用索引提升查询效率
3. **数据完整性**：使用外键约束保证数据一致性
4. **扩展性**：使用JSONB存储灵活的游戏状态
5. **实时性**：支持实时数据同步

## 索引设计

### 索引策略

1. **主键索引**：所有表都有主键索引
2. **外键索引**：所有外键字段都有索引
3. **唯一约束索引**：确保数据唯一性
4. **查询优化索引**：基于常用查询模式创建复合索引
5. **部分索引**：对特定条件创建部分索引

### 性能索引

```sql
-- 房间查询优化
CREATE INDEX idx_rooms_status_created ON rooms(status, created_at DESC);

-- 房间成员查询优化
CREATE INDEX idx_room_members_room_seat ON room_members(room_id, seat_no);

CREATE INDEX idx_chat_messages_unread ON chat_messages(receiver_uid, is_read)
```

### 用户管理函数

#### update_user_status(user_status text)

更新用户在线状态。

```sql
CREATE OR REPLACE FUNCTION update_user_status(user_status text)
RETURNS boolean AS $$
BEGIN
  UPDATE public.profiles
  SET 
    status = user_status,
    last_online_at = now()
  WHERE uid = auth.uid();
  
  RETURN true;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

#### search_users(search_term text, limit_count int)

搜索用户。

```sql
CREATE OR REPLACE FUNCTION search_users(search_term text, limit_count int DEFAULT 20)
RETURNS TABLE (
  uid uuid,
  nickname text,
  avatar_url text,
  status text,
  level int,
  total_games int,
  win_rate numeric
) AS $$
BEGIN
  RETURN QUERY
  SELECT 
    p.uid,
    p.nickname,
    p.avatar_url,
    p.status,
    p.level,
    p.total_games,
    p.win_rate
  FROM public.profiles p
  WHERE p.uid != auth.uid()
    AND p.nickname ILIKE '%' || search_term || '%'
  ORDER BY 
    CASE WHEN p.status = 'online' THEN 0 ELSE 1 END,
    p.nickname
  LIMIT limit_count;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

### 好友系统函数

#### accept_friend_request(request_id uuid)

接受好友请求。

```sql
CREATE OR REPLACE FUNCTION accept_friend_request(request_id uuid)
RETURNS boolean AS $$
DECLARE
  v_sender_uid uuid;
  v_receiver_uid uuid;
  v_status text;
BEGIN
  -- 获取请求详情
  SELECT sender_uid, receiver_uid, status
  INTO v_sender_uid, v_receiver_uid, v_status
  FROM public.friend_requests
  WHERE id = request_id;
  
  -- 检查请求是否存在且为待处理状态
  IF NOT FOUND THEN
    RAISE EXCEPTION 'Friend request not found';
  END IF;
  
  IF v_status != 'pending' THEN
    RAISE EXCEPTION 'Friend request is not pending';
  END IF;
  
  -- 检查接收者是否为当前用户
  IF auth.uid() != v_receiver_uid THEN
    RAISE EXCEPTION 'You can only accept requests sent to you';
  END IF;
  
  -- 更新请求状态
  UPDATE public.friend_requests
  SET status = 'accepted'
  WHERE id = request_id;
  
  -- 创建好友关系（确保一致性排序）
  INSERT INTO public.friends (user1_uid, user2_uid)
  VALUES (
    LEAST(v_sender_uid, v_receiver_uid),
    GREATEST(v_sender_uid, v_receiver_uid)
  )
  ON CONFLICT DO NOTHING;
  
  RETURN true;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

#### get_user_friends()

获取用户好友列表。

```sql
CREATE OR REPLACE FUNCTION get_user_friends()
RETURNS TABLE (
  friend_uid uuid,
  nickname text,
  avatar_url text,
  status text,
  last_online_at timestamptz,
  level int,
  total_games int,
  win_rate numeric,
  created_at timestamptz
) AS $$
BEGIN
  RETURN QUERY
  SELECT 
    CASE 
      WHEN f.user1_uid = auth.uid() THEN f.user2_uid 
      ELSE f.user1_uid 
    END AS friend_uid,
    p.nickname,
    p.avatar_url,
    p.status,
    p.last_online_at,
    p.level,
    p.total_games,
    p.win_rate,
    f.created_at
  FROM public.friends f
  JOIN public.profiles p ON (
    (f.user1_uid = auth.uid() AND p.uid = f.user2_uid) OR
    (f.user2_uid = auth.uid() AND p.uid = f.user1_uid)
  )
  WHERE f.user1_uid = auth.uid() OR f.user2_uid = auth.uid()
  ORDER BY p.last_online_at DESC;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

### 聊天系统函数

#### send_message(target_uid uuid, message_content text)

发送聊天消息。

```sql
CREATE OR REPLACE FUNCTION send_message(target_uid uuid, message_content text)
RETURNS TABLE (
  message_id uuid,
  room_id uuid,
  sender_uid uuid,
  receiver_uid uuid,
  content text,
  is_read boolean,
  created_at timestamptz
) AS $$
DECLARE
  v_room_id uuid;
  v_user1_uid uuid;
  v_user2_uid uuid;
  v_message_id uuid;
BEGIN
  -- 确保一致性排序
  v_user1_uid := LEAST(auth.uid(), target_uid);
  v_user2_uid := GREATEST(auth.uid(), target_uid);
  
  -- 获取或创建聊天房间
  SELECT id INTO v_room_id
  FROM public.chat_rooms
  WHERE user1_uid = v_user1_uid AND user2_uid = v_user2_uid;
  
  -- 如果房间不存在，创建它
  IF v_room_id IS NULL THEN
    INSERT INTO public.chat_rooms (user1_uid, user2_uid)
    VALUES (v_user1_uid, v_user2_uid)
    RETURNING id INTO v_room_id;
  END IF;
  
  -- 插入消息
  INSERT INTO public.chat_messages (room_id, sender_uid, receiver_uid, content)
  VALUES (v_room_id, auth.uid(), target_uid, message_content)
  RETURNING id INTO v_message_id;
  
  -- 更新房间的最后消息时间
  UPDATE public.chat_rooms
  SET last_message_at = now()
  WHERE id = v_room_id;
  
  -- 返回消息
  RETURN QUERY
  SELECT 
    cm.id AS message_id,
    cm.room_id,
    cm.sender_uid,
    cm.receiver_uid,
    cm.content,
    cm.is_read,
    cm.created_at
  FROM public.chat_messages cm
  WHERE cm.id = v_message_id;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

#### get_user_chat_rooms()

获取用户聊天房间列表。

```sql
CREATE OR REPLACE FUNCTION get_user_chat_rooms()
RETURNS TABLE (
  room_id uuid,
  other_user_uid uuid,
  other_user_nickname text,
  other_user_avatar_url text,
  other_user_status text,
  last_message_content text,
  last_message_at timestamptz,
  unread_count int
) AS $$
BEGIN
  RETURN QUERY
  SELECT 
    cr.id AS room_id,
    CASE 
      WHEN cr.user1_uid = auth.uid() THEN cr.user2_uid 
      ELSE cr.user1_uid 
    END AS other_user_uid,
    p.nickname AS other_user_nickname,
    p.avatar_url AS other_user_avatar_url,
    p.status AS other_user_status,
    (
      SELECT cm.content 
      FROM public.chat_messages cm 
      WHERE cm.room_id = cr.id 
      ORDER BY cm.created_at DESC 
      LIMIT 1
    ) AS last_message_content,
    cr.last_message_at,
    (
      SELECT COUNT(*)
      FROM public.chat_messages cm
      WHERE cm.room_id = cr.id
        AND cm.receiver_uid = auth.uid()
        AND cm.is_read = false
    ) AS unread_count
  FROM public.chat_rooms cr
  JOIN public.profiles p ON (
    (cr.user1_uid = auth.uid() AND p.uid = cr.user2_uid) OR
    (cr.user2_uid = auth.uid() AND p.uid = cr.user1_uid)
  )
  WHERE cr.user1_uid = auth.uid() OR cr.user2_uid = auth.uid()
  ORDER BY cr.last_message_at DESC;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

### 游戏管理函数

#### submit_turn(p_game_id uuid, p_action_id uuid, p_expected_turn_no int, p_payload jsonb)

提交游戏回合。

```sql
CREATE OR REPLACE FUNCTION submit_turn(
  p_game_id uuid,
  p_action_id uuid,
  p_expected_turn_no int,
  p_payload jsonb
)
RETURNS TABLE (
  turn_no int,
  current_seat int,
  status text,
  rankings int[]
) AS $$
DECLARE
  v_current_turn_no int;
  v_current_seat int;
  v_game_status text;
BEGIN
  -- 获取当前游戏状态
  SELECT turn_no, current_seat, status
  INTO v_current_turn_no, v_current_seat, v_game_status
  FROM public.games
  WHERE id = p_game_id;
  
  -- 检查回合号是否匹配
  IF v_current_turn_no != p_expected_turn_no THEN
    RAISE EXCEPTION 'turn_no_mismatch';
  END IF;
  
  -- 检查游戏状态
  IF v_game_status != 'playing' THEN
    RAISE EXCEPTION 'game_not_playing';
  END IF;
  
  -- 插入回合记录
  INSERT INTO public.turns (game_id, turn_no, seat_no, action_id, payload)
  VALUES (p_game_id, v_current_turn_no + 1, v_current_seat, p_action_id, p_payload);
  
  -- 更新游戏状态
  UPDATE public.games
  SET 
    turn_no = v_current_turn_no + 1,
    current_seat = (v_current_seat + 1) % 4,
    updated_at = now()
  WHERE id = p_game_id;
  
  -- 返回更新后的状态
  RETURN QUERY
  SELECT turn_no, current_seat, status, rankings
  FROM public.games
  WHERE id = p_game_id;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

### 数据类型说明

#### UUID

用于唯一标识符，确保全局唯一性。

#### JSONB

用于存储复杂数据结构，支持高效的JSON操作。

#### TIMESTAMPTZ

带时区的时间戳，确保时间准确性。

#### TEXT

用于变长字符串，适合存储用户输入。

#### CHECK约束

确保数据符合业务规则。

```sql
CHECK (seat_no BETWEEN 0 AND 3)
```

确保数据唯一性。

```sql
UNIQUE (room_id, seat_no)
```
