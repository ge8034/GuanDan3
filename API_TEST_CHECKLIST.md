# API测试检查清单

**版本**: v1.0
**创建日期**: 2026-03-19
**负责人**: API测试团队
**适用范围**: 所有API端点开发和质量保证

---

## 📋 概述

本检查清单用于确保所有API端点开发符合项目开发标准，防止类似讨论问题中的异步逻辑、事件系统和状态管理问题再次发生。

### 使用时机
- API端点开发前设计评审
- 代码实现过程中自查
- 代码审查时检查
- 测试执行前验证
- 部署前质量门禁

---

## 🧪 异步逻辑检查清单

### 1. useEffect依赖完整性检查
- [ ] **依赖数组完整**: 所有在useEffect回调中使用的变量都包含在依赖数组中
- [ ] **稳定引用**: 使用useCallback和useMemo创建稳定的函数和值引用
- [ ] **清理函数**: 所有需要清理的资源都有清理函数
- [ ] **执行次数**: useEffect执行次数符合预期，无意外重复执行

### 2. 事件监听器管理检查
- [ ] **添加和移除配对**: 所有事件监听器都有对应的移除操作
- [ ] **无限循环防护**: 事件处理函数不会触发自身导致无限循环
- [ ] **内存泄漏防护**: 组件卸载时清理所有事件监听器
- [ ] **错误边界**: 事件处理函数有错误捕获和处理

### 3. 异步操作时序检查
- [ ] **时序依赖**: 异步操作之间的时序依赖明确
- [ ] **竞态条件防护**: 使用abort controller或类似机制防止竞态条件
- [ ] **超时处理**: 所有异步操作都有合理的超时设置
- [ ] **错误恢复**: 异步操作失败后有恢复机制

### 4. 状态管理检查
- [ ] **状态一致性**: 相关状态保持一致性，无矛盾状态
- [ ] **状态更新时序**: 状态更新时序符合业务逻辑
- [ ] **状态同步**: 客户端状态与服务器状态同步机制健全
- [ ] **状态持久化**: 必要状态有持久化机制

## ⚡ 性能测试检查清单

### 1. 响应时间检查
- [ ] **P95达标**: 响应时间P95 ≤ 100ms
- [ ] **P99达标**: 响应时间P99 ≤ 200ms
- [ ] **平均响应**: 平均响应时间在合理范围内
- [ ] **基线对比**: 与性能基线对比无显著下降

### 2. 吞吐量检查
- [ ] **并发支持**: 支持20个并发用户
- [ ] **峰值处理**: 能处理每秒5次操作的峰值
- [ ] **资源使用**: CPU使用率 ≤ 30%（20并发）
- [ ] **内存使用**: 内存使用在合理范围内

### 3. 稳定性检查
- [ ] **错误率**: 错误率 < 0.1%

- [ ] **耐久测试**: 通过24小时耐久测试
- [ ] **恢复能力**: 故障后能快速恢复

### 4. 监控指标检查
- [ ] **指标收集**: 收集P50、P90、P95、P99响应时间
- [ ] **错误监控**: 监控错误率和错误类型
- [ ] **资源监控**: 监控CPU、内存、数据库连接
- [ ] **告警设置**: 设置适当的性能告警

## 📝 使用说明

### 1. 检查频率
- **开发过程中**: 每日自查相关检查项
- **代码审查时**: 审查人检查所有相关检查项
- **测试执行前**: 测试人员检查测试相关检查项
- **部署前**: 部署负责人检查所有检查项

### 2. 检查记录
- 每次检查记录检查日期、检查人、检查结果
- 发现问题记录问题描述、严重程度、修复计划
- 检查结果存档备查

### 3. 问题处理
- 高优先级问题必须修复后才能继续
- 中优先级问题制定修复计划
- 低优先级问题记录待后续处理

### 4. 持续改进
- 定期评审检查清单的有效性
- 根据项目经验更新检查项
- 优化检查流程和工具
